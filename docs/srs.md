## Στόχος του συστήματος 
Ανάπτυξη μιας διαδικτυακής συνεργατικής πλατφόρμας διαχείρισης έργων λογισμικού. Η πλατφόρμα θα επιτρέπει σε ομάδες να υλοποιούν πολλά έργα λογισμικού με βάση τη μεθοδολογία Scrum, υποστηρίζοντας τις βασικές έννοιες των Backlogs, Sprints και User Stories καθώς και τις σχετικές αναφορές. Χρήση του μοντέλου «Λογισμικό ως Υπηρεσία» (Software as a Service – SaaS).

## Εμβέλεια συστήματος
Το σύστημα θα προορίζεται για άτομα και ομάδες που θέλουν να οργανώσουν την διαχείριση ενός έργου με βάση την Scrum μεθοδολογία. Το σύστημα παρέχει τις βασικές λειτουργίες διαχείρισης έργου με σαφήνεια και καθαρότητα, ώστε οι χρήστες να επικεντρώνονται στην γρήγορη και αποτελεσματική αποπεράτωση του έργου τους.


## Απαιτήσεις
Εδώ περιγράφονται οι λειτουργικές απαιτήσεις του συστήματος. Αυτές είναι ταξινομημένες σε ομάδες με βάση την ρόλο τους στην εφαρμογή. Η κάθε ομάδα έπειτα κατηγοριοποιείται σε σελίδες οι οποίες διαθέτουν τις εξειδικευμένες λειτουργίες της εφαρμογής. Κάθε απαίτηση σελίδας έχει έναν σύνδεσμο ο οποίος μεταβαίνει στον πίνακα προδιαγραφών της απαίτησης. Εκεί εξηγείται αναλυτικότερα καθώς και ο τρόπος εκτέλεσής της.

### Παραδοχές
  * Ο Project Owner ορίζεται αυτόματα ως ο Product Owner.
  * Tο μοναδικό δικαίωμα που θα έχει παραπάνω ο Project Owner, Product Owner και δεν θα μπορούν αυτήν την διαδικασία να την κάνουν τα απλά μέλη θα είναι να δημιουργεί User Stories στο Backlog και να μπορεί να διαγράψει το project ή να αλλάξει το όνομά του. 
  * Το Sprint θα μπορεί να ενεργοποιηθεί μια φορά και κάθε φορά στο project θα υπάρχει μόνο ένα ενεργό Sprint.
  * Tα Sprints θα περιέχουν μόνο Tasks και η διάρκειά τους δεν θα επηρεάζουν το Sprint.

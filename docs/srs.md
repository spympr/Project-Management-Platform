## Στόχος του συστήματος 
Ανάπτυξη μιας διαδικτυακής συνεργατικής πλατφόρμας διαχείρισης έργων λογισμικού. Η πλατφόρμα θα επιτρέπει σε ομάδες να υλοποιούν πολλά έργα λογισμικού με βάση τη μεθοδολογία Scrum, υποστηρίζοντας τις βασικές έννοιες των Backlogs, Sprints και User Stories καθώς και τις σχετικές αναφορές. Χρήση του μοντέλου «Λογισμικό ως Υπηρεσία» (Software as a Service – SaaS).

## Εμβέλεια συστήματος
Το σύστημα θα προορίζεται για άτομα και ομάδες που θέλουν να οργανώσουν την διαχείριση ενός έργου με βάση την Scrum μεθοδολογία. Το σύστημα παρέχει τις βασικές λειτουργίες διαχείρισης έργου με σαφήνεια και καθαρότητα, ώστε οι χρήστες να επικεντρώνονται στην γρήγορη και αποτελεσματική αποπεράτωση του έργου τους.


## Απαιτήσεις
Εδώ περιγράφονται οι λειτουργικές απαιτήσεις του συστήματος. Αυτές είναι ταξινομημένες σε ομάδες με βάση την ρόλο τους στην εφαρμογή. Η κάθε ομάδα έπειτα κατηγοριοποιείται σε σελίδες οι οποίες διαθέτουν τις εξειδικευμένες λειτουργίες της εφαρμογής. Κάθε απαίτηση σελίδας έχει έναν σύνδεσμο ο οποίος μεταβαίνει στον πίνακα προδιαγραφών της απαίτησης. Εκεί εξηγείται αναλυτικότερα καθώς και ο τρόπος εκτέλεσής της.

#### Σελίδα Kanban board (Project Homepage)
##### Προεπισκόπηση ενεργών tasks του χρήστη για το project
| Απαίτηση           | Προεπισκόπηση ενεργών tasks του χρήστη για το project        |
| ------------------ | ------------------------------------------------------------ |
| **Σελίδα**         | Kanban board (Project Homepage)                              |
| **Προδιαγραφή**    | H σελίδα Kanban board θα είναι η αρχική σελίδα την οποία ο χρήστης βλέπει μόλις συνδεθεί σε κάποιο project του. Με τον Πίνακα Kanban ένας χρήστης, έχει ξεκάθαρη εικόνα σε τι κατάσταση βρίσκεται τα Tasks που του έχουν ανατεθεί. Ο χρήστης πατώντας σε ένα task θα μπορεί να ενημερωθεί για την κατάσταση του task αλλά και να ανανεώσει την κατάστασή του ο ίδιος ο χρήστης. O χρήστης έχοντας επιλέξει να επεξεργαστεί ένα Task, θα οι πληροφορίες που θα λαβμάνει θα είναι το ποια μέλη συνεισφέρουν σ' αυτό, εκτιμώμενη διάρκεια, σε ποιο User Story ανήκει, αλλά και σε ποια από τις τρεις κατηγορίες (Εκκρεμεί, Σε εξέλιξη, Ολοκληρώθηκε) ανήκει.  |


#### Σελίδα Πίνακας Scrum
##### Προβολή και επεξεργασία του συνολικού Product Backlog
| Απαίτηση           | Προβολή και επεξεργασία του συνολικού Product Backlog        |
| ------------------ | ------------------------------------------------------------ |
| **Σελίδα**         | Πίνακας Scrum                                                |
| **Προδιαγραφή**    | Στον Πίνακα Scrum παρουσιάζεται το Backlog στο οποίο έχει προσθέσει ο Product Owner τα User Stories, αλλά και τα Sprints, όπου σε καθένα ξεχωριστά φαίνεται η κατανομή των Tasks. Πιο ειδικά ο Product Owner θα μπορεί να δημιουργεί στο Backlog User Stories. Όταν τα δημουργεί ο Product Owner θα μπορεί να δίνει το όνομα, μια περιγραφή και το αν θα είναι Epic ή Issue. Αυτά θα υπάρχουν στην στήλη του Backlog. Επίσης οποιοσδήποτε θα μπορεί με full text search με το όνομα του user story να βρει το αντίστοιχο.   |
